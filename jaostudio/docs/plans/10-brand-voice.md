# Brand Voice & Copywriting System

## Voice Definition

```
Tone:     Confident · Clear · Technical but accessible · Minimal · Business-aware
Persona:  High-end product engineer who also understands business
Feeling:  "This person could build my entire online presence and I'd trust them completely"
```

## Always Avoid

| Avoid | Why |
|---|---|
| Hype language ("revolutionary", "cutting-edge") | Sounds like marketing agency, not engineer |
| Buzzwords ("synergy", "leverage", "optimize") | Corporate fluff, zero personality |
| Fake agency tone ("we believe", "our mission") | You're one person — own it |
| Self-deprecation ("just a developer", "trying to learn") | Undermines premium positioning |
| Tech jargon without context | Alienates business owners |
| Lengthy explanations | Premium sites show more, say less |

## Allowed Vocabulary

| Category | Preferred Words |
|---|---|
| What you do | Build, engineer, develop, design, create, craft |
| Quality | Premium, modern, fast, responsive, clean, polished |
| Outcome | Conversion, trust, credibility, clarity, growth |
| Process | Architecture, strategy, system, structure, workflow |
| Your role | Developer, engineer, designer, builder |
| Clients | Businesses, founders, teams, companies |

## Copy Patterns

### Headlines

```
Pattern: [What] engineered/built for [outcome]
Example: Modern websites engineered for performance, trust, and conversion.

Pattern: [Outcome] websites that [benefit]
Example: High-performance websites that make businesses look premium.
```

### Section Introductions

```
Selected Work
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Premium websites and interfaces built for modern businesses.

Each project demonstrates a focus on performance, clarity,
and conversion-driven design.
```

### Service Descriptions

```
Landing Pages
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
High-conversion landing pages optimized for speed,
responsiveness, and modern user experience.

Best for: product launches, campaigns, SaaS signups
```

### About Section

```
I specialize in building modern websites
that help businesses establish stronger
credibility online.

My focus extends beyond visuals —
I prioritize performance, responsiveness,
maintainability, and conversion-focused UX.
```

## What the Copy Must Communicate

Every piece of copy should pass this filter:

```
Does this sound like a high-end engineer
who understands business,
or like every other freelancer?
```

If it could appear on a generic portfolio template, rewrite it.

## Copy Length Rules

| Element | Max Length |
|---|---|
| Headline | 15 words (usually 6-10) |
| Tagline | 12 words |
| Section description | 30 words |
| Project description | 60 words |
| Service card | 20 words |
| CTA text | 4 words |
| Button text | 4 words |

Shorter is almost always better. Premium sites say less and show more.

## The "Business Value" Translation

Whenever describing a technology or feature, include the business reason:

```
❌ "Built with Next.js and Tailwind"
✅ "Built with Next.js for fast page loads and SEO"

❌ "Responsive design"
✅ "Looks premium and works perfectly on every device"

❌ "Framer Motion animations"
✅ "Smooth, intentional animations that guide users naturally"
```
