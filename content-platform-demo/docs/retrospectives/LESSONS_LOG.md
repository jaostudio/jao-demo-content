# Lessons Log

## Purpose

Running log of lessons learned across projects. Each entry documents what went wrong, what rule was added, and whether it prevented recurrence.

## Entries

| Date | Project | Issue | Rule Added | File Updated | Prevented Recurrence? |
|---|---|---|---|---|---|
| | | | | | |

## How To Add An Entry

After each retrospective:

1. Identify the root cause of each significant issue.
2. Determine whether a rule, checklist item, or test would prevent recurrence.
3. Add the rule to the appropriate library file.
4. Add an entry to this log.
5. Verify on the next project that the rule was followed.
