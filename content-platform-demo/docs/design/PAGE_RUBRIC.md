# Page Rubric

## Scoring Scale

| Category | Weight |
|---|---|
| Visual hierarchy | 15 |
| Brand consistency | 15 |
| Responsive behavior | 15 |
| Interaction quality | 10 |
| Copy clarity | 10 |
| Accessibility | 10 |
| Performance | 10 |
| Conversion clarity | 10 |
| Technical cleanliness | 5 |
| **Total** | **100** |

## Thresholds

```
90+  Ready for launch
80–89 Ship only if non-critical issues
70–79 Needs polish before review
<70  Not portfolio or client ready
```

## Per-Route Checklist

### Homepage

```
- [ ] Clear promise above fold
- [ ] Primary CTA visible
- [ ] Secondary CTA non-competing
- [ ] Brand impression strong within 3 seconds
- [ ] Trust signal present (proof, social, or guarantee)
- [ ] No generic SaaS-template feel
- [ ] Mobile layout preserves hierarchy
- [ ] Loads within 3 seconds on 3G
```

### Portfolio

```
- [ ] Crisp screenshots
- [ ] Featured project immediately visible
- [ ] Case study has proof (decisions, constraints, outcomes)
- [ ] Technical decisions explained
- [ ] Reviewer path obvious (click → see → understand)
- [ ] Live demo links work (or marked as demo)
- [ ] Mobile layout preserves image quality
```

### Pricing / Quote Flow

```
- [ ] Step clarity — user knows where they are
- [ ] Estimate confidence — clearly what they get
- [ ] Validation clarity — errors are specific and fixable
- [ ] Submit success reassurance — confirmation and next steps
- [ ] Mobile completion path — no barriers on small screens
- [ ] Admin handoff reliability — data is persisted
```

### Dashboard

```
- [ ] Information hierarchy — most important data first
- [ ] Empty states — useful guidance when no data exists
- [ ] Loading states — skeleton or progress indicator
- [ ] Error recovery — clear messages and retry options
- [ ] Permission boundaries — users see only what they should
- [ ] Data density — readable without overwhelming
- [ ] Keyboard accessibility — all actions reachable via keyboard
```

### Automation Tool

```
- [ ] Run status visible — user knows if it's running, done, or failed
- [ ] Failure reason visible — clear error explanation
- [ ] Manual retry available — user can restart failed tasks
- [ ] Logs accessible — user can inspect what happened
- [ ] Side effects clear — user knows what the automation does
- [ ] Audit trail present — history of actions
```
