# Copy Guide

## Voice and Tone

| Attribute | Description |
|---|---|
| Voice | [Authoritative / Friendly / Technical / Warm / Direct / Playful] |
| Tone | Adjusts by context: helpful in docs, direct in errors, warm in onboarding |
| Audience | [Describe the reader — their knowledge level, expectations, and context] |

## Writing Principles

```
1. Use specific nouns and concrete outcomes. Avoid vague abstractions.
2. Every claim must be backed by proof (screenshot, metric, decision, example).
3. Short sentences for actions. Longer sentences for explanation. Vary rhythm.
4. Remove adjectives that do not add information.
5. Opinionated positioning beats neutral corporate language.
6. Write error and empty states with the same care as marketing copy.
7. Use active voice. "You can export your data" not "Data can be exported."
```

## Headings

### Rules

```
- Every heading must be specific to this project.
- If a heading could be copied into another project unchanged, rewrite it.
- Headings should contain at least one specific noun or verb.
```

### Examples

```
Bad:   "Solutions for Modern Business"
Good:  "Quote Workflows for Service Businesses"

Bad:   "Our Mission"
Good:  "Help Roofers Send Better Proposals in Half the Time"
```

## CTAs

### Rules

```
- CTA text must be specific to the action.
- Use verbs that describe what happens next.
- Avoid "Submit" for everything. Use "Send Proposal", "Start Free Trial", "View Case Study".
```

## Error Messages

### Structure

```
[What went wrong] + [Why it happened (if helpful)] + [How to fix it]
```

### Examples

```
Bad:   "An error occurred."
Good:  "Could not save your quote because the session expired. Please refresh and try again."

Bad:   "Invalid input."
Good:  "Enter a valid email address (e.g., name@example.com)."
```

## Empty States

### Structure

```
[What this space is for] + [What to do next]
```

### Examples

```
Good:  "No projects yet. Create your first project to start tracking quotes."
Good:  "No results match this filter. Try a different search term or clear filters."
```

## Success States

### Structure

```
[What happened] + [What to do next]
```

### Examples

```
Good:  "Quote sent to client. They will receive an email with a link to review and approve."
Good:  "Proposal published. Share this link with your client: [link]"
```

## Proof Language

```
- Instead of "Trusted by leading companies" → name specific companies or contexts.
- Instead of "Built for performance" → "Pages load in under 2 seconds on 3G."
- Instead of "Enterprise-grade security" → "Data encrypted at rest and in transit. SOC 2 compliant."
```

## Anti-Patterns

```
- "Elevate your business"
- "Seamless experience"
- "Unlock your potential"
- "Tailored solutions"
- "Empowering creators"
- "Transform your workflow"
- Overuse of "designed to help you..."
- Generic benefit stacks
- Every paragraph having the same rhythm
- Vague testimonials without specific outcomes
```
